# Анализ шаблона ТЗ и структуры заполнения

## 1. Структура шаблона Typst

### Основные файлы:
- **`main.typ`** - точка входа, запускает компиляцию
- **`template.typ`** - шаблон оформления страниц (титульный лист, колонтитулы, стили)
- **`cfg.typ`** - конфигурация документа (формирует данные из `data.typ`)
- **`data.typ`** - **ОСНОВНОЙ ФАЙЛ ДЛЯ ЗАПОЛНЕНИЯ** - содержит метаданные проекта
- **`body.typ`** - **ОСНОВНОЙ ФАЙЛ ДЛЯ ЗАПОЛНЕНИЯ** - содержит текст всех разделов ТЗ

### Принцип работы:
1. В `data.typ` заполняются базовые данные (название, участники, руководитель)
2. В `body.typ` заполняется содержание всех разделов ТЗ
3. `main.typ` собирает всё вместе и генерирует PDF

## 4. Информация из description.txt для заполнения

### Аннотация проекта:
> Sentinel — интеллектуальный планировщик с мультимодальным вводом. Пользователь кидает скрин письма, фото списка или короткий текст, система извлекает задачи, дедлайны, приоритеты и формирует таймблоки с экспортом в Календарь. Свои модели не нужны, распознавание и интерпретация идут через внешние Vision/LLM API.

### Ключевые особенности:
- Модель Resource Battery (индикатор энергии дня)
- Auto-Rest Nudges (микро-перерывы)
- Achievements (достижения за паттерны)
- Rebalance (оптимизация порядка дел)

### Цель проекта:
> Реализовать мобильное приложение "Sentinel"

## 9. Важные замечания

### Особенности Typst синтаксиса:
- `=` - заголовок 1 уровня
- `==` - заголовок 2 уровня
- `===` - заголовок 3 уровня
- `#data.project_name` - вставка переменной из data.typ
- Таблицы создаются через `#figure(table(...))`
- Для выделения используется `*жирный*` и `_курсив_`

### Автоматически заполняемые элементы:
- Титульный лист (из data.typ)
- Лист утверждения (из data.typ)
- Содержание (автоматически из заголовков)
- Нумерация страниц
- Колонтитулы с кодом проекта

### Что НЕ нужно заполнять:
- Лист регистрации изменений (пустой шаблон)
- Аннотация (автоматически генерируется в cfg.typ)
- Список ГОСТов (уже есть стандартный)
